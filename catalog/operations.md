# operations plugin

**Marketplace:** `knowledge-work-plugins/operations`
**Status in this repo:** cataloged only — not yet installed locally.

## Description

Optimize business operations: vendor management, process documentation, change management, capacity planning, and compliance tracking. Useful when you're formalizing how the business actually runs so it doesn't live in someone's head.

## Install

```bash
claude plugins add knowledge-work-plugins/operations
```

After install, copy the plugin folder into `./plugins/operations` in this repo (see README for the extraction pattern).

## Skills

| Skill                   | When to use                                                                                  |
| ----------------------- | -------------------------------------------------------------------------------------------- |
| `capacity-plan`         | Quarterly planning, team feels overallocated, you need utilization forecasts                |
| `change-request`        | Proposing a system or process change that needs approval — includes impact + rollback plan  |
| `compliance-tracking`   | Track requirements and audit readiness for SOC2, HIPAA, etc.                                |
| `process-doc`           | Formalize a process: flowcharts, RACI matrix, SOPs                                          |
| `process-optimization`  | Analyze a process and propose improvements                                                  |
| `risk-assessment`       | Identify, assess, and mitigate operational risks                                            |
| `runbook`               | Document a recurring on-call or ops procedure                                               |
| `status-report`         | Weekly/monthly leadership update with KPIs, risks, action items                             |
| `vendor-review`         | Evaluate a vendor: cost analysis, risk assessment, recommendation                           |
